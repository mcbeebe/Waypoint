Executive Summary

Waypoint 2.0 builds on 12 completed sprints of the GAS MVP and React Native app foundation. This PRD defines 69 remaining sprints (S0A–S0F + S13–S81) across 12 phases to deliver all 11 strategic features identified in the March 2026 Strategic Analysis. Phase 0 addresses critical AI Navigator hardening gaps; Phases 1–9 deliver core features; Phase 10 adds Resources & Blog; Phase 11 (conditional) enables Epic MyChart integration.

Current Product Status

The waypoint-app already includes:

  • Supabase Auth — Apple Sign-In, Google Sign-In (Calendar + Gmail scopes), Email/Password
  • 6-Step Onboarding — Family setup, child profiles, diagnosis tags, RC selection, contact info
  • AI Navigator — RAG-powered chat with pgvector embeddings, streaming, tone calibration (collaborative → assertive → adversarial)
  • Action Plan Tracker — 4-state management (proposed/in-progress/completed/abandoned) with offline queue
  • Calendar & Deadlines — Appointment scheduling, deadline tracking, push notifications
  • Document Vault — Supabase Storage integration, file upload/download, document metadata
  • Internationalization — English, Spanish, Vietnamese (es, vi, en)
  • QA & Stability — ErrorBoundary, Sentry integration, Toast system, NetworkBanner

Strategic Goals for Waypoint 2.0

Deploy the 11 strategic features below to create a comprehensive, defensible platform that parents can't live without:

  1. AI Navigator Hardening (Phase 0) — Critical bug fixes: Opus 4.6 upgrade, source attribution, RAG fallbacks, diagnosis context, legal disclaimers
  2. Calendar + Google Sync — Bi-directional sync with Google Calendar
  3. Financial Tracker — Expense management with tax deduction reports
  4. Provider & Service Management — Care team directory, service authorizations
  5. IEP Document Intelligence — PDF OCR, goal parsing, weakness analysis, improvement suggestions
  6. Community Forum — Diagnosis-specific forums, peer support, DMs
  7. Co-Parent Coordination — Multi-user family sharing, task assignment, activity feed
  8. Provider Portal (B2B) — Separate experience for therapists, attorneys, coordinators
  9. Resources & Guides — Curated content library, blog, difficulty-leveled articles
  10. Blog — In-app blog with news, tips, legal updates, community stories
  11. Epic MyChart Integration (Conditional) — SMART on FHIR OAuth to import medical records and sync with Navigator

Plus Data Moat instrumentation (anonymous outcomes tracking) and launch prep (TestFlight, App Store, accessibility audit).

Development Model

Each sprint is scoped for 5–10 minutes of AI-assisted development using Cowork sessions. All sprints include rigorous acceptance criteria and automated test plans. This doc is both a contract and a test suite.

Feature Requirements

Calendar + Google Sync Phase 1

Complete the partially-built calendar by adding bi-directional Google Calendar sync. OAuth tokens are already stored in SecureStore. Google Calendar and Gmail scopes already requested during sign-in.

User Stories

  • As a parent, I want my Waypoint appointments to appear in my Google Calendar so I don't double-book
  • As a parent, I want to see my existing Google Calendar events in Waypoint so I have one unified view
  • As a parent, I want automatic sync so I don't have to manually update two calendars

Key Requirements

  • Bi-directional sync (Waypoint ↔ Google Calendar)
  • Conflict resolution (Waypoint event wins if both sides modified)
  • Visual indicator for synced vs local-only events
  • Token refresh handling on 401 responses
  • Handles network errors gracefully

Sprint Range: S25–S30 (6 sprints)

IEP Document Intelligence Phase 4

The killer differentiator. AI reads uploaded IEP PDFs, extracts goals, flags weaknesses, suggests improvements, and generates meeting prep materials. No competitor offers this.

User Stories

  • As a parent, I want to upload my child's IEP and have AI explain what each goal means
  • As a parent, I want AI to flag weak or vague goals so I know what to challenge
  • As a parent, I want specific suggestions for stronger goal language citing IDEA requirements
  • As a parent, I want to compare this year's IEP to last year's to see progress
  • As a parent, I want a meeting prep doc generated before my IEP meeting

Key Requirements

  • PDF OCR/text extraction via Supabase Edge Function
  • Goal parsing (domain, baseline, target, measurement, timeline)
  • Weakness analysis (vague goals, missing baselines, unrealistic timelines) with severity rating
  • Improvement suggestions with IDEA statute citations and rewritten goal text
  • Side-by-side comparison of two IEPs with diff highlighting
  • Meeting prep doc generation (customized to RC/district)
  • Save improved goals as action items
  • Caching of analysis results

Sprint Range: S74–S76 (3 sprints)

Epic MyChart Integration Phase 11 (Conditional)

Connect to Epic MyChart via SMART on FHIR OAuth to import medical records (diagnoses, medications, labs, appointments) and inject into AI Navigator context. IEP/educational records NOT accessible via FHIR (FERPA, not HIPAA). Focus is medical records only.

User Stories

  • As a parent, I want Waypoint to automatically know my child's diagnoses from their medical records
  • As a parent, I want my child's medical appointments to appear in Waypoint's calendar
  • As a parent, I want the AI Navigator to reference my child's actual medications and conditions when giving advice

Key Requirements

  • SMART on FHIR patient-facing OAuth flow with MyChart redirect
  • Access token storage and secure refresh handling
  • FHIR R4 read-only access (Condition, Medication, Appointment, Observation, DocumentReference resources)
  • Auto-population of diagnoses and medications with manual override option
  • Calendar sync with visual indicators for MyChart vs local appointments
  • Health Records screen showing labs, clinical notes, and allergy information
  • Privacy consent shown before health data shared with AI
  • Disconnect capability to revoke MyChart access
  • Graceful handling when MyChart not connected (no blocking)

Note: This phase is conditional on securing a health system partnership. Sandbox development is free; production deployment requires $15K–$200K+ per health system.

Technical Architecture Notes

Stack Overview

  • Frontend: React Native 0.76 + Expo 52 + TypeScript (strict mode)
  • State Management: React hooks + Supabase real-time subscriptions
  • Navigation: React Navigation (native-stack + bottom-tabs)
  • Backend: Supabase Postgres + Auth + Storage + Edge Functions
  • AI: Anthropic Claude API (Sonnet for fast queries, Opus for complex analysis)
  • RAG: pgvector embeddings (text-embedding-3-small, 1536 dims) with hybrid semantic + keyword search
  • Offline: AsyncStorage queue for CRUD operations, auto-sync on reconnect
  • Auth: Apple Sign-In, Google Sign-In (Calendar + Gmail scopes), Email/Password
  • Design System: Custom tokens in src/lib/theme.ts (colors: navy, teal, coral, sage; spacing scale; radii)

API Integration Points

  • Google Calendar API: Bi-directional sync via OAuth tokens stored in SecureStore
  • Gmail API: Read inbox, send drafts, create/update messages
  • Anthropic Claude API: Via Supabase Edge Function proxy for cost control and request logging
  • Document OCR: Edge Function (integrated with document processing service)

Database Schema (11 Tables)

Core tables already exist: families, children, diagnoses, providers, services, documents, expenses, appointments, deadlines, chat_sessions, chat_messages. Additional tables added per phase: forum_categories, forum_threads, forum_posts, forum_reactions, forum_reports, direct_messages, family_members, analytics_events, provider_profiles, knowledge_embeddings (pgvector).

Performance Targets

  • Cold app start: < 3 seconds
  • Screen transitions: < 300ms
  • AI response start: < 2 seconds (streaming)
  • API caching: 5 minutes default
  • Image lazy loading: All images off-screen

Cost Optimization

  • Prompt Caching: Cache system prompts and KB context in Claude API calls to reduce token costs
  • Streaming: Stream all AI responses to reduce perceived latency and improve UX
  • Edge Functions: Batch requests where possible, use Sonnet for fast queries, Opus only for complex analysis
  • Data Aggregation: Weekly refresh of analytics insights to avoid real-time computation

Testing Strategy

Unit Tests

Each hook gets comprehensive unit tests with mocked Supabase client. Target coverage: 80% of hooks. Frameworks: Jest (or Vitest) for JavaScript, React Native Testing Library for components.

  • Test all CRUD operations in isolation
  • Mock API responses and network errors
  • Verify offline queue behavior
  • Test data transformation and filtering

Integration Tests

Each screen tested with React Native Testing Library. Verify rendering, user interactions, navigation flows, and state changes.

  • Render screen and verify UI elements present
  • Simulate user actions (tap, scroll, text input)
  • Verify navigation to child screens
  • Test loading states and error handling

End-to-End Tests

Critical flows tested with Detox or Maestro. Flows: Onboarding, AI Chat, Action Management, Document Upload, Calendar Sync.

  • Test from real device (or simulator)
  • Verify full flow including API calls
  • Test offline→online transitions
  • Test multi-language flows

Acceptance Testing

Each sprint's acceptance criteria must be verified before marking the sprint complete. Create a test checklist per sprint, grouped by acceptance criterion.

Regression Testing

Sprint 70 runs full regression across all features. Test on iPhone SE (small screen) and iPhone 15 Pro Max (large screen). Test all 3 languages (en, es, vi). Test offline mode.

Performance Testing

  • Measure cold start time (adb logcat for Android, Instruments for iOS)
  • Measure screen transition times (React Navigation performance monitoring)
  • Measure AI response time (measure time to first token, not full response)
  • Profile memory usage under load

Accessibility Testing

WCAG AA compliance verified with automated tools + manual screen reader testing.

  • Run automated accessibility scanner on all screens
  • Test with VoiceOver (iOS) and TalkBack (Android)
  • Verify 44px touch targets
  • Verify color contrast ratios (4.5:1 for text)
